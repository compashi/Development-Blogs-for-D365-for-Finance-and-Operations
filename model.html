Models in Microsoft Dynamics 365 for Finance and Operations

A model in D365 is a group of elements, which comprises of metadata and source files, altogether constitute to a distributable software solution. Each model must belong to a deployable package (collection of models and its binary and resource files).

Models can be of 2 types. One, that is deployed in its own package, and the other that is a part of existing package. A model that is deployed in its package is used to create new metadata elements and to extended business logic of referred models. The later on the other hand is used to work on extensions or overlaying existing out-of-the box customizations.

Creating a model

To create a model:

  • Go to menu Dynamics 365 > Model Management > Create Model
  • Set parameters, model's layer, and then click Next
  • If you selected for a model to be deployed on its own package, choose the models that you are going to refer to for that model and click Finish
  • Else, select a model in the drop down, for this to make a part of it, and then click Finish
  • Verify the folder has been created in C:\AOSServices\PackagesLocalDirectory\<Package_Name>\<Model_Name> (Note that <Package_Name> will only occur if you select the option for the model ot be a part of existing package)

Updating model parameters

To update model parameters:

  • Go to menu Dynamics 365 > Model Management > Update Model
  • Select the model that you want to update
  • Update its metadata properties (Note that you cannot modify model's layer) and click Next
  • Check models that need to be referred, or uncheck models that don't need to be referred in the selected model
  • Click Finish and verify that model parameters have been updated (if they don't, Go to menu Dynamics 365 > Model Management > Refresh Models or restart Visual Studio)

Deleting a model

Deleting a model can be divided into two ways. One for deleting a model that belongs to its own package and the later one belonging to a package with multiple models. both of them can be deleted in the same way possible, but its recommended by Microsoft to utilize ModelUtil.exe explained below (Note that these practices are only applicable development or test environment, doing so on a live production server can lead to loss of data or worse).

Deleting model using ModelUtil.exe (recommended)

  • Run command prompt as administrator
  • Navigate to C:\AOSServices\PackagesLocalDirectory\Bin
  • Type command ModelUtil.exe /? to list commands that can be executed using model utility
  • To delete model, type the command ModelUtil.exe -delete -metadatastorepath="<Path>" -modelname="<Model_Name>"
  • Confirm that you want to delete it, by typing Y in command prompt
  • Close command prompt and verify that model folder isn't there in its specified folder

Deleting model from local directory:

  • Stop AOS Web Service and Batch Management Service
  • Delete the model folder located at C:\AOSServices\PackagesLocalDirectory\<Package_Name>\<Model_Name>
  • Restart AOS Web Service and Batch Management Service
  • Refresh models in Visual studio via Dynamics 365 > Model Management > Refresh Models
  • Perform full database synchronization in Visual Studio via Dynamics 365 > Synchronize Database
