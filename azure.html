Azure Active Directory Application Integration with Microsoft Dynamics 365 for Finance and Operations

With the release of D356FO, there has been an immense architecture shift from server-client architecture (AX2012) to cloud based architecture (D365). Therefore, Integrating 3rd party applications has to be done in another way to ensure data flow to and from outside world applications.

Software-as-a-Service (SaaS) providers and application/service developers can develop cloud based services or applications that can be consumed by D356FO via secure authorization and sign-in service provided by azure active directory (AAD).

To Integrate AAD application with D356FO:

  • Go to azure portal
  • Click on Azure Active Directory
  • Click App Registrations
  • Click New application registration
  • Add parameters for you application
  • After application is registered, the page would look like
  • Click Required Permissions
  • Select Microsoft Dynamics ERP in the API section
  • Select all permissions and roles
  • After permissions are created, the success message will look like
  • Click Grand Permissions
  • Copy Application Id
  • Go to D365 main url and navigate to Modules > System administration > Azure Active Directory applications
  • Set the client Id as the application Id that was copied back in step#12

You are now done with setting up AAD native application, so that you can integrate with D356FO. There are many operations that can be performed such as import/export, but will be demonstrated in Recurring Integration Services (RIS) blog.
